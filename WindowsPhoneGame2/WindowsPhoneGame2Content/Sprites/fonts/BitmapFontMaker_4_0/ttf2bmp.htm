TrueType to Bitmap Converter (ttf2bmp.exe)

This utility converts Windows TrueType fonts into .bmp files that can be imported using the XNA Framework Content Pipeline, and converted into SpriteFont objects by BitmapFontProcessor.

Overview

When you add a new font description to an XNA Framework project, this creates an XML description file describing what font you want to use. The Content Pipeline reads this description, renders out the appropriate characters using the Windows Font API, and converts them into a format that can be efficiently rendered using the SpriteFont class.

That is all very well for drawing standard Windows fonts, but some games will want a more specialized look. Perhaps you want an outline on your characters, or a color gradient, or some kind of crazy embossed letters formed out of multicolored flowers with flames licking around the base. For these situations, the XNA Framework provides BitmapFontProcessor. This lets you draw whatever character shapes you like into a specially formatted image, which can then be converted into a SpriteFont by the pipeline.

The only problem is, drawing an entire font by hand is really hard! That's where this utility comes in. It does basically the same thing as the regular font pipeline, letting you choose a font and then rendering out each character into a bitmap, but it saves the resulting bitmap into a regular .bmp file. You can take this .bmp into your paint application and edit it however you like, using filters to apply special effects, and then feed the modified .bmp back into the pipeline for use in your game.

Using the Tool

When you run the utility, you will see a dialog box that lets you choose what font, size, and style you want to export. This also includes a character setting, which defaults to the range 0x20 through 0x7F, which is the standard ASCII character range. This means your font will include all of the regular English alphabet. If you want to include other characters—for example, accented vowels for European languages, or Japanese characters—you will need to modify these values.

When you click the Export button, you will be prompted to select an output .bmp file, and the font will be rendered out. If you load the resulting .bmp file into a paint application, you will notice that it contains a set of letters separated by a pink marker color. This pink is important because the pipeline uses it to detect where each character image is located. If you edit the bitmap, you must take care not to accidentally change the pink marker areas at the same time. The bitmap also contains an alpha channel, which you will need to preserve and edit along with the RGB color data.

When you add the font .bmp to your XNA Game Studio Express project, it defaults to using the Texture - XNA Framework content importer, and the Texture (Sprite, 32bpp) - XNA Framework content processor. (You can find these settings in the Visual Studio properties pane after selecting the font in Solution Explorer.) This default importer is fine, but you will need to change the processor to Sprite Font Texture - XNA Framework. Build your project, and the ContentManager will be ready to load your custom SpriteFont.
